# KHB オンライン採点シート — ユーザーガイド

俳句甲子園のオンライン審査・採点を行うためのシステムです。  
管理者・審査員・配信担当の 3 つの役割を想定しています。

---

## 1. システム概要

| 画面 | 対象 | URL |
|------|------|-----|
| 管理パネル | 管理者 | `admin.html` |
| 試合管理 | 管理者 | `admin-match.html` |
| 審査員管理 | 管理者 | `admin-judges.html` |
| 審査員入力画面 | 審査員 | `judge.html?token=<トークン>` |
| スコアボード（横型） | OBS 配信 | `obs-scoreboard.html?venue=<会場コード>` |
| スコアボード（縦型） | OBS 配信 | `obs-scoreboard-vertical.html?venue=<会場コード>` |
| 勝数オーバーレイ | OBS 配信 | `winnum_obs_overlay.html?venue=<会場コード>` |

---

## 2. 事前準備（管理者）

### 2.1 会場の作成

複数会場で同時開催する場合、会場（venue）を事前に作成します。  
デフォルトでは `default` 会場が用意されています。

### 2.2 審査員の登録

**審査員管理画面（`admin-judges.html`）** で審査員を登録します。

1. 審査員名を入力
2. Voice Key を入力（読み上げ音声の識別用、任意）
3. 「追加」ボタンを押す

登録すると **トークン**（例: `khb-a1b2c3d4e5f6...`）が発行されます。  
このトークンは **審査員ごとに 1 つ**で、全試合・全会場共通で使用します。

> **重要**: トークンは審査員に安全に配布してください。  
> トークンを知っていれば誰でもその審査員として採点できます。

### 2.3 試合の作成

**試合管理画面（`admin-match.html`）** で試合を登録します。

| 項目 | 説明 |
|------|------|
| 試合コード | 一意の識別子（例: `A-1`, `B-2`） |
| 試合名 | 表示名（例: `予選A 1試合目`） |
| 紅チーム名 | 紅（赤）側のチーム名 |
| 白チーム名 | 白側のチーム名 |
| 対戦数 | 通常は `5`（先鋒〜大将）。3 人制の場合は `3` |
| タイムライン | 同時進行グループ番号（後述） |
| 会場 | この試合が行われる会場 |

### 2.4 期待審査員の割り当て

試合管理画面 または 審査員管理画面で、各試合にどの審査員を割り当てるかを設定します。  
割り当て順（sort_order）がスコアボード上の表示順になります。

---

## 3. タイムライン（同時進行管理）

**タイムライン（timeline）** は、試合の進行順序を管理する数値です。

- 同じ timeline 値 → 同時に進行する試合（並行試合）
- 小さい値 → 先に進行する試合

**例:**

| 試合 | timeline |
|------|----------|
| A-1（予選A 1試合目） | 1 |
| B-1（予選B 1試合目） | 1 |
| A-2（予選A 2試合目） | 2 |
| B-2（予選B 2試合目） | 2 |
| 決勝 | 3 |

- A-1 と B-1 は同時進行（timeline=1）
- 両方終わったら A-2 と B-2 へ（timeline=2）
- 最後に決勝（timeline=3）

審査員が複数の試合に割り当てられている場合、以下の優先度で自動的に案内されます:

| 優先度 | 状態 | 記迳 |
|--------|------|------|
| 高 | `accepting=true`、未完了 | 通常の受付中 |
| 中 | `accepting=false`、未完了 | 受付停止中 |
| 低 | 完了済み | 試合終了・講評中 |

同優先度内では timeline の小さい試合が優先されます。

## 4. 試合の運用フロー

### 4.1 試合開始

1. **管理パネル（`admin.html`）** を開く
2. 管理用シークレットを入力
3. 会場をプルダウンから選択
4. 試合をプルダウンから選択
5. **「この試合を開始」** ボタンを押す

これにより、その会場の `current_match_id` が設定され、審査員の画面にこの試合が表示されます。

### 4.2 対戦の進行（E1〜E6 フロー）

1 試合は複数の対戦（先鋒戦〜大将戦）で構成されます。  
各対戦は以下のイベントで進行します:

```
試合開始 → [対戦 1] → [対戦 2] → ... → [対戦 5] → 試合終了
```

各対戦の流れ:

| ステップ | イベント | 操作者 | 説明 |
|----------|----------|--------|------|
| 1 | — | 管理者 | **受付開始**（accepting=true）— 試合開始時は自動でオン |
| 2 | **E1** | 審査員 | 初回採点を提出 |
| 3 | **E2** | 審査員 | 修正がある場合に再提出（任意、何度でも可） |
| 4 | **E3** | 自動 | 全審査員の提出が揃った（e3_reached=true） |
| 5 | **E5** | 管理者 | **「E5 確定」ボタン** → スナップショット保存・受付停止・勝敗決定 |
| 6 | **E6** | 管理者 | **「E6 次の対戦へ」ボタン** → epoch+1・受付再開 |

> **E5 と E6 の間**に、得点読み上げや結果確認を行います。

### 4.3 管理パネルの状態表示

管理パネルの上部にリアルタイムで以下が表示されます:

| 表示 | 意味 |
|------|------|
| `accepting: true` / `false` | 審査員からの提出を受け付けているか |
| `e3_reached: true` / `false` | 全審査員が提出済みか |
| `current_match_id: true` / `false` | 選択中の試合が現在の試合か |
| `state.epoch: N` | 現在の対戦番号 |

### 4.4 審査員の並び替え

管理パネルの「審査員の並び替え」セクションで、ドラッグ＆ドロップで表示順を変更できます。  
変更後に **「並び順を保存」** ボタンを押すと、スコアボードの表示順が更新されます。

## 5. 審査員の操作

### 5.1 審査員画面へのアクセス

配布されたトークン付き URL でアクセスします:

```
judge.html?token=khb-xxxxxxxxxxxxxxxxxxxx
```

### 5.2 採点の入力

1. 画面に試合名・対戦名・チーム名が表示されます
2. **紅（赤）** と **白** それぞれについて:
   - **作品点**: 1〜10 の整数
   - **鑑賞点**: 0・1・2 のいずれか
3. 合計点と旗（◆）は自動計算されます
4. **「送信」** ボタンで提出

### 5.3 受付状態の表示

画面上部に現在の受付状態が表示されます:

| 表示 | 意味 |
|------|------|
|（何も表示されない） | 通常の受付中 |
| `現在は受付停止中です` | E5確定後など、管理者が一時的に受付を停めている状態 |
| `試合終了（講評中）` | 全対戦の確定が完了し、次の試合に移行するまでの講評期間 |

`受付停止中`・`試合終了` のいずれの場合も、画面に表示されたデータ（試合名・採点内訳等）はそのまま保持されます。

### 5.4 修正

- 送信後も受付中（accepting=true）であれば **「修正する」** ボタンで再入力できます
- 修正は何度でも可能です（revision が増加）

### 5.5 旅の自動判定

- 合計点が高い方に旅（◆）がつきます
- 合計点が同じ場合は、**作品点**が高い方に旅がつきます

### 5.6 次の対戦・次の試合

- 管理者が E6（次の対戦へ）を実行すると、画面が自動更新されて次の対戦の入力画面になります
- 試合の全対戦が確定した後は `試合終了（講評中）` と表示され、最終採点内訳が画面に保持されます
- 次の timeline の試合に割り当てられている場合は、管理者が次試合を開始したときに自動的にそちらに切り替わります
- **URL の変更は不要です**（1 つのトークンで全試合を通して使用）

## 6. OBS 配信設定

### 6.1 スコアボード

OBS のブラウザソースに以下の URL を設定します:

| レイアウト | URL |
|----------|-----|
| 横型 | `obs-scoreboard.html?venue=<会場コード>` |
| 縦型 | `obs-scoreboard-vertical.html?venue=<会場コード>` |

- `?venue=default` — デフォルト会場
- `?venue=A` — A 会場
- 会場コードを省略すると `default` になります

**表示制御**: 管理パネルの得点読み上げ時に自動でスコアボードが表示/非表示されます（`scoreboard_visible` フラグ）。

### 6.2 勝数オーバーレイ

試合全体の勝数（例: 紅 2 - 白 1）を表示するオーバーレイです:

```
winnum_obs_overlay.html?venue=<会場コード>
```

- E5（確定）実行時に自動更新されます
- 透過背景で OBS 上に重ねて使用します

---

## 7. 得点読み上げ

管理パネルの「得点読み上げパネル」で、確定した得点を音声で読み上げることができます。

1. E3 到達（全審査員提出完了）後、再生キューが自動生成されます
2. **「開始〜終了 全員読み上げ」** ボタンで再生開始
3. 再生中はスコアボードが自動表示されます
4. 再生完了後、スコアボードは自動非表示になります

---

## 8. 複数会場の運用

### 8.1 基本構成

- 各会場に独立した **state**（epoch/accepting/current_match_id）
- 試合は **venue_id** で会場に紐付け
- 審査員トークンは会場をまたいで共通

### 8.2 運用例

**3 会場同時開催の場合:**

1. 会場 A, B, C を venues に登録
2. 各会場の試合を作成（venue_id を指定）
3. 各会場に 1 台の管理 PC + 管理パネルを用意
4. 各管理パネルで会場を選択して運用
5. OBS は `?venue=A`, `?venue=B`, `?venue=C` でそれぞれ配信

### 8.3 会場間の制約

- ある会場の管理者が、**別の会場の試合を開始することはできません**（venue_id の一致チェック）
- 審査員は複数会場の試合に割り当て可能（timeline で順序管理）
